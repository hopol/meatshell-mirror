# 贡献指南

感谢您对本项目的关注！欢迎通过以下方式参与贡献。

## 如何贡献

### 1. 报告问题

如果您发现同步过程中存在问题，请：

1. 查看现有的 Issue，避免重复
2. 创建新的 Issue，详细描述问题
3. 包括错误信息、复现步骤和环境信息

### 2. 提交改进

如果您有改进建议或代码修复：

1. Fork 本仓库
2. 创建您的特性分支：`git checkout -b feature/amazing-feature`
3. 提交您的更改：`git commit -m '添加某个特性'`
4. 推送到分支：`git push origin feature/amazing-feature`
5. 创建一个 Pull Request

### 3. 改进文档

文档的改进同样重要，包括：

- 修复错别字
- 补充说明
- 添加示例
- 翻译成其他语言

## 开发环境

### 环境要求

- Git 2.0 或更高版本
- GitHub 账户
- 基本的命令行操作能力

### 本地测试

1. 克隆仓库
2. 修改同步脚本
3. 在本地测试同步流程
4. 确认无误后提交更改

## 代码规范

### 提交信息

- 使用中文编写提交信息
- 格式：`<类型>: <描述>`
- 类型包括：feat（新功能）、fix（修复）、docs（文档）、style（格式）、refactor（重构）、test（测试）、chore（构建/工具）

示例：
```
feat: 添加标签同步功能
fix: 修复同步失败问题
docs: 更新使用说明
```

### 代码风格

- 保持代码简洁易读
- 添加必要的注释
- 遵循项目的编码规范

## 审核流程

1. 提交 Pull Request
2. 等待自动检查通过
3. 等待维护者审核
4. 根据反馈进行修改
5. 审核通过后合并

## 行为准则

- 尊重所有参与者
- 使用友善和包容的语言
- 接受建设性的批评
- 专注于对社区最有利的事情

## 联系方式

如有任何问题，请通过以下方式联系：

- 创建 Issue
- 发送邮件至维护者

感谢您的贡献！
